# Roster & saturation tracker — coverage across the captive user set

**Status:** READY (worked instance — two sizes)
**Loaded when:** BETWEEN interviews, as you plan who to talk to next and decide when to stop.

For an internal tool you don't recruit from a funnel — you have N named, captive users. The job is the
opposite of consumer research: not to find enough people, but to **cover the roles that matter and
stop before you over-sensitize the same operators into parroting your framing.** You build the roster
from the role-interest map, propose the next interviewee, flag role gaps, and call saturation when a
fresh interview stops producing any new workaround, term, or system.

Pick the size that fits the engagement. Don't run the campaign machinery on a five-person tool.

---

## Size A — Spot-check (small or short engagement)

For a tool with a handful of operators, or a sprint too short for cadence. One table, no ceremony.

| Person | Role | Talked to? | New signal? |
|---|---|---|---|
| | operator | | |
| | commissioner | | |
| | third role (auditor / admin / downstream) | | |

**Stop rule:** every distinct role has at least one interview, and the last one turned up nothing new.
That's enough — don't manufacture a cadence.

---

## Size B — Campaign (larger set, work spread over weeks)

For a wider user set interviewed over time.

**Roster** — named people mapped to role and the week you plan to talk to them.

| Person | Role | Metric measured on | Week | Done |
|---|---|---|---|---|

**Coverage matrix** — are the roles that matter actually covered, or are you over-weighting the easy-
to-reach ones?

| Role | # interviewed | Still unheard |
|---|---|---|
| operator | | |
| commissioner | | |
| third role(s) | | |

**Saturation note** — refreshed after each interview:

> *Last interview new signal: yes / no. New this round: [workaround / term / system, or "none"].
> Still unheard: [downstream consumer, auditor, …]. Recommend: [one more in role X / saturated, stop].*

**Stop rule:** fresh signal has stopped across the roles that matter — not when you've simply talked to
everyone. Cap and rotate rather than re-interviewing the same operator until they echo you back.

---

**Tie-in:** the roster reads from `modules/people/role-interest-map.md` for the role set, and feeds
the BEFORE beat — "five operators interviewed, zero commissioners, zero auditor" is the signal that
the next guide should target the gap.
