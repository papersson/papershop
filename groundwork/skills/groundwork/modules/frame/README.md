# frame — converge: one framed problem, one measurable success line

Mid-level index (progressive disclosure). Enter when you have pain from several interviews and a build decision is coming, and the question is no longer "what's broken" but "which one thing do we build, and how will we know it worked."

## Leaves
- `problem-frame.md` — narrow all the captured pain to ONE framed problem plus a hard operational success line, kept re-openable as a gate. [READY]
- `opportunity-tree.md` — the dual-rooted opportunity tree: a cross-interview synthesis container rooted at BOTH the commissioner's metric and operator friction, so the people split stays visible in the artifact. [READY]

## Fit
The forward-motion convergence after the CORE understanding branches (as-is, people, domain). Those branches diverge — they accumulate workarounds, tensions, terms, source-of-truth gaps. frame is where that pile becomes a build decision: the tree organizes the divergent evidence and keeps whose-interest visible; the problem-frame picks one branch of it and writes the measurable line you'll be held to. Feed it the snapshots and the role map; hand its output to `modules/slice/` to test and ship thin. Refresh the tree across the campaign; close the frame once before any build commitment.
